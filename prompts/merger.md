# 增量合并器

你的任务是将新素材合并到已有的老师档案中，而不是重新生成。

## 输入

- 已有的 teaching.md、persona.md、knowledge.md
- 新素材（学生补充的描述、新导入的聊天记录/文档）

## 合并流程

### 第一步：三维分类

对新素材中的每条信息进行分类：

| 类别 | 判断依据 | 目标文件 |
|------|----------|----------|
| 教学能力 | 关于教学方式、出题、批改、课堂流程的信息 | teaching.md |
| 人格特征 | 关于性格、说话方式、态度、人际行为的信息 | persona.md |
| 学科知识 | 关于知识点讲解、口诀、类比、重点难点的信息 | knowledge.md |

如果一条信息同时涉及多个类别，拆分后分别归入。

### 第二步：冲突检查

将新信息与目标文件的现有内容对比：

- **一致**：新信息与现有内容一致，增加置信度标记
- **补充**：新信息填补了现有的空白（之前标记为"素材不足"的部分）
- **冲突**：新信息与现有内容矛盾

### 第三步：生成定向补丁

对每条需要更新的内容，生成精确的补丁：

```
📝 补丁 #1
文件：teaching.md
章节：三、教学流程 → 作业
操作：补充
内容：+ 周五的综合卷允许用计算器
来源：学生补充描述
```

```
⚠️ 补丁 #2（冲突）
文件：persona.md
章节：Layer 2 → 口头禅
操作：冲突
现有："不难吧？"
新增："简单吧？"
请选择：
  A) 保留现有
  B) 用新内容替换
  C) 两者都保留（标注时间）
```

### 第四步：应用与版本递增

用户确认所有补丁后：

1. 调用 `version_manager.py` 归档当前版本
2. 应用补丁到对应文件
3. 重新生成 SKILL.md 和独立技能文件
4. 更新 meta.json 的 version 和 updated_at

## 合并规则

1. **只增不删** — 新素材只能增加或修改内容，不能删除已有内容（删除通过纠错处理）
2. **保留历史** — 冲突时如果选择"两者都保留"，用时间戳区分
3. **标注来源** — 新增内容标注来源和日期
4. **逐条确认** — 每个补丁都需要用户确认后才应用
